# 产品展示网页 — 从零到完整的阶梯式计划

## 总体路线图

```
第1步      第2步       第3步        第4步        第5步        第6步
静态页面 → 产品卡片 → 交互动画 → 视频弹窗 → 产品系列分区 → 展示页完成
                                                              ↓
                                                           第7步
                                                      后台管理系统
                                                              ↓
                                                           第8步
                                                          部署上线
```

**核心原则**：
- 每一步都产生一个"能用"的网页，不是半成品
- 每完成一步，你用浏览器打开就能看到效果
- 每一步只新增一项能力，不会跳跃
- 先把展示页做到完美，最后再加后台

## 第 1 步：创建第一个静态网页（约 30 分钟，难度 ⭐）

### 目标
创建一个最简单的 HTML 网页，浏览器打开能看到标题和文字。

### 你会学到的
- 了解项目文件夹结构
- 知道如何用浏览器打开网页
- 看懂 HTML 的基本结构（标签、标题、段落）

### 产出的效果
打开网页后看到：公司名称 + 一句话介绍 + 一张图片

### 涉及的文件
只有一个 `index.html`

### 你需要准备
- 公司名称
- 一张产品图片（放在文件夹里）

> **这步的意义**：确认工具链（VS Code + 浏览器）能正常工作。如果你这步做不下去，后面都不用开始。

---

## 第 2 步：美化页面样式（约 30 分钟，难度 ⭐）

### 目标
给第 1 步的页面加上颜色、字体、布局，让它好看。

### 你会学到的
- CSS 是什么（给网页"化妆"）
- 颜色、间距、排版的基础概念
- 怎么让内容居中、分列

### 产出的效果
一个有设计感的公司介绍页：深色/浅色背景、好看的字体、居中的布局

### 涉及的文件
- `index.html`（修改）
- 新增 `style.css`

## 第 11 步：数据分离——用 JSON 管理产品信息（约 1 小时，难度 ⭐⭐）

### 目标
把产品信息从 HTML 里抽出来，放到一个单独的 JSON 数据文件里。页面加载时自动读取 JSON 并渲染产品卡片。

### 你会学到的
- JSON 是什么（一种存数据的格式，像 Excel 表格一样有行有列）
- JavaScript 如何读取数据文件
- 数据和页面分离的好处（改数据不用改代码）

### 产出的效果
- 新增一个 `products.json` 文件，里面存所有产品信息
- 新增产品：在 JSON 文件里加一条数据 → 页面自动多一个卡片
- **为后面的后台系统打基础**

### 涉及的文件
- 新增 `products.json`
- 修改 `script.js`
- `index.html` 里产品卡片的 HTML 由 JS 动态生成，不需要手写

> **这步是重要转折点**：从此以后，加新产品不需要改 HTML 代码，改 JSON 数据就行。

## 第 12 步：引入 Vue.js——让开发更高效（约 1.5 小时，难度 ⭐⭐⭐）

### 目标
把之前的纯 HTML/CSS/JS 网页改成用 Vue.js 框架来实现。功能不变，但代码更干净、后续加功能更方便。

### 你会学到的
- Vue.js 是什么（一个让网页开发更简单的工具）
- 组件概念（像乐高积木，每个组件是一块独立的积木）
- 数据绑定（数据变了，页面自动跟着变，不用手动操作）

### 产出的效果
功能和第 11 步完全一样，但代码结构更好，为后台系统做好准备

> **为什么需要这步？** 纯 JS 写到后面会越来越乱。Vue.js 是搭大型项目的"脚手架"，有了它后面加后台会轻松很多。

### 涉及的文件

> **注意**：这步是纯前端的后台演示。数据存在浏览器本地（localStorage），关掉浏览器数据还在，但换一台电脑就看不到了。真正的后端在第 14 步。

## 关键讨论点记录

### 讨论点 1：第 1-11 步用纯 HTML，还是直接学 Vue？

| 方案 | 优点 | 缺点 |
|------|------|------|
| **A：先纯 HTML（第1-11步），再转 Vue（第12步）** ✅ 已选 | 入门简单，每步改动小，容易理解 | 第12步需要重写，有重复工作 |
| **B：直接上 Vue（跳过第1-11步的纯HTML）** | 不用重复写代码，一路到底 | 开头学习曲线稍陡 |

### 讨论点 2：产品数据在 JSON 文件还是直接写数据库？

- **JSON 文件阶段（第11步）**：更新产品 = 改一个文本文件，适合暂时不需要后台的情况
- **数据库阶段（第14步）**：更新产品 = 在后台网页填表单，适合日常维护

两个阶段都保留。JSON 文件的好处是第 11 步就能用上数据驱动的模式，不需要等到第 14 步。

### 讨论点 3：先做后台还是先做完展示页？

**先做完展示页（第1-12步），最后加后台（第13-14步）**。理由：
- 展示页是你的核心交付物（客户看到的）
- 后台是辅助工具（你自己用的）
- 先让展示页好看、能用，再考虑管理效率
